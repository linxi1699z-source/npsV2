# 支付后台 Axure 原型设计与交互规范

来源：`/Users/xiemin/Documents/Axure/支付后台原型页面.zip` 的 Axure HTML 导出包。本文用于后续按现有后台风格生成新原型页面、交互说明、Axure 实现清单或 Axure 风格 HTML。

## 1. 适用范围

- 适用于支付后台、会员/用户、订单、产品、权益、支付渠道、多语言、IAP 同步等管理类页面。
- 优先使用 Axure 支持的元件和交互：矩形、文本、输入框、下拉框、复选框、表格、动态面板、中继器、树形导航、热区、图片、线段、内置动作。
- 生成 `.rp` 时只使用 Axure 原生能力，不依赖自定义前端框架或不可落地的 JS。生成 HTML 时可以复刻 Axure 导出结构和样式，但交互语义仍按 Axure 动作设计。

## 2. 页面骨架

### 2.1 画布和基础布局

- 基准页面容器：白底，常见可视区域约 `1479 x 956`。
- 页面 `body` 不随大表格无限扩展。新增管理页默认以 `1479px` 或业务指定固定宽度承载主操作区；列表/表格外层容器固定宽度，表格列总宽超过容器时，在列表容器内显示横向滚动条。
- 左侧导航栏固定宽 `150px`，高度 `956px`，背景 `#F2F2F2`，边框 `#D7D7D7`。
- 主内容区从 `x=160-172px` 起排布。顶部横线常用 `x=172, y=46, w=1300, h=1`，颜色 `#D7D7D7`。
- 品牌文字 `Ringconn` 放在左上角，约 `x=32, y=25, w=84, h=21`，Arial Bold，18px 左右。

### 2.2 导航规则

- 必须保留左侧树形导航，根节点为 `支付中心`。
- 一级业务导航优先包括：产品管理、权益管理、订单管理、支付渠道管理、会员/用户管理。
- 新增功能的层级判断：
  - 独立业务域或高频入口：作为一级导航，如“退款管理”“优惠券管理”。
  - 依附某个业务域的配置或详情：作为二级页面，通过表格操作按钮或页签进入，如“产品折扣信息”“补充多语言”“同步 IAP 产品”。
  - 临时任务、批处理、导入导出：优先做成当前页面按钮或弹窗，不新增左侧导航。
- 导航选中态优先使用底部蓝色线条，不使用黄色填充作为默认选中态。一级导航和二级导航均可采用 `#1E98D7` 底线，线高 `2-3px`，放在导航文本下方；线宽跟随文字宽度，不应超过文字长度。未选中保持透明或灰底。仅在需要复刻旧 Axure 页面时，才使用浅黄高亮 `rgba(255,255,128,0.8235)`。
- 二级导航必须相对一级导航缩进，建议二级项距左侧导航内容起点 `32-40px`，或比一级文本起点多 `16-22px`，形成清晰树形层级。
- 导航文本使用 `PingFang SC Regular`，12px。确认版页面可保留 `[确认]` 或 `[已确认]` 前缀；新设计稿如无特殊评审状态，建议去掉此前缀。

### 2.3 面包屑和页签

- 主内容顶部使用面包屑：`支付中心 / 当前模块 / 子页面`，常放在主内容左上，字体 `PingFang SC Regular` 13px。
- 页签使用矩形 `box_2`：
  - 单个页签常见 `150 x 40`。
  - 未选中背景 `#F2F2F2`。选中页签可保持同背景，不做填充色变化，仅增加 semibold 字重。
  - 选中页签底部优先加蓝色横线作为强调，颜色 `#1E98D7`，高度 `2-3px`，宽度跟随页签文字，不应超过文字长度。
  - 页签切换使用动态面板 `Set Panel State`，同时设置选中态。

## 3. 字体、颜色和间距

### 3.1 字体

- 全局 Axure 默认：`ArialMT, Arial, sans-serif`，13px，`#333333`。
- 中文业务文本优先：`PingFangSC-Regular, PingFang SC, sans-serif`，13px。
- 中文强调、表头、主要按钮：`PingFangSC-Semibold, PingFang SC Semibold, PingFang SC, sans-serif`，字重 650。
- 英文、ID、数字混排可沿用 Arial。中文和英文混排时可分别设置 span 字体，保持原型中的混排习惯。
- 说明区或导航可使用 12px；日期选择器和极小辅助文本可使用 10px。

### 3.2 颜色

- 主文字：`#333333` 或 `#000000`。
- 表格正文常见深灰：`#3F3F46`。
- 辅助/禁用文字：`#999999`、`#AAAAAA`。
- 链接文字：`#0000FF` 或 Axure 默认链接蓝 `#1E98D7`，可加下划线。
- 边框：`#D7D7D7`。
- 页面/表格正文背景：`#FFFFFF`。
- 次级背景：`#F2F2F2`。
- 禁用输入框背景：`#F0F0F0`。
- 主要按钮背景：`#7F7F7F`，文字白色，圆角 `5px`。
- 次级或禁用按钮可用 `#AAAAAA`、`#D7D7D7`。

### 3.3 间距

- 控件内部文字 padding 通常为 `2px`。
- 搜索表单控件高度 `32px`，标签高度 `18px`。
- 表格行高默认 `40px`；部分说明表或复杂行可用 `45px`、`80px`。
- 弹窗标题距左约 `15px`，距上约 `20-23px`。标题下方分割线距上约 `61px`。
- 弹窗按钮区域通常在底部，确认和取消按钮间距约 `20-33px`。

## 4. 页面元素规范

### 4.1 文本和标签

- 普通正文：13px，左对齐。
- 搜索项描述文本：右对齐，宽度按文本长度取 `44-76px` 或同组统一宽度。
- 表单弹窗中的字段标签：右对齐，输入控件放在标签右侧。
- 必填字段前加 `*`，如 `* 权益名称`。
- 表格内容默认居中对齐；长文本、说明类字段可左对齐，但需显式标注。

### 4.2 输入框

- 搜索表单输入框默认 `200 x 32`，边框 `1px #D7D7D7`，白底，文字左对齐，字体 13px。
- 搜索条件整体按栅格右侧对齐：标签右对齐，控件紧跟标签右侧。
- 输入框内部 padding 保持 `2px`，占位提示使用 `#999999`。
- 控件状态：
  - 默认：白底，边框 `#D7D7D7`。
  - 鼠标移入：边框高亮为 `#AAAAAA`。在 Axure 中用 `MouseOver` 交互样式实现。
  - 获取焦点：边框高亮为 `#1E98D7`，可加 `1px` 蓝色外框或保持原边框变色。用 `Focused` 交互样式或 `OnFocus` 设置样式实现。
  - 输入中：保持焦点态，文本颜色 `#000000`。
  - 失焦：有值时恢复默认边框并保留值；无值时恢复占位提示。
  - 禁用：背景 `#F0F0F0`，文字 `#999999` 或 `#000000`，边框 `#D7D7D7`，不可触发焦点和 hover 高亮。
  - 错误：边框 `#D9001B`，错误提示放在控件下方或右侧，文字 `#D9001B`，13px；再次输入或重新校验通过后恢复默认/焦点态。
- 必填输入框的校验顺序：先校验空值，再校验格式，再校验重复或业务规则。错误提示文案使用 `{字段名}不能为空。`、`{字段名}格式不正确。`、`{字段名}已经存在。`。

#### 4.2.1 日期范围选择器

- 日期范围必须使用单个 RangePicker 触发控件承载开始和结束时间。触发控件内部按 Ant Design RangePicker 类似结构展示两个文本段：`Start date` / `End date` 或业务中文占位，中间使用箭头或短横分隔。不要用两个分离的日期输入框作为搜索区默认形态。
- 占位示例日期只作为 placeholder，不等同于默认选中区间。除非业务明确要求默认时间范围，否则打开日期面板时不应自动选中占位日期。
- 触发框默认宽度 `240-280px`，高度 `32px`，白底，边框 `#D7D7D7`，右侧放日历 icon 或下拉箭头；hover/focus/禁用/错误态同普通输入框。
- 点击触发框或 icon 后显示日期范围动态面板。面板默认隐藏，显示时位于触发框下方；业务只需单月时用单月日历，需要跨月选择时优先用左右双月日历，双月面板宽度建议 `620-680px`。
- 面板内不再使用“开始/结束”两个原生日期输入框作为主交互。开始和结束日期应在同一个日历面板内点击选择：第一次点击设置开始日期，第二次点击设置结束日期；若第二次日期早于第一次，则自动交换开始/结束。
- 选择过程使用临时变量保存。第一次点击设置开始日期，第二次点击设置结束日期；结束日期选中后立即把 `range_start`、`range_end` 写入隐藏值/中继器筛选值，更新触发框两段文本，并关闭面板。
- 已选区间必须高亮：
  - 开始日、结束日使用主色实心或深色底，白色文字。
  - 开始和结束之间的日期使用浅色连续背景，形成可识别的区间带。
  - hover 日期使用浅灰底或边框高亮，不覆盖已选区间样式。
  - 今日可使用细边框或浅色标记，但优先级低于选中态。
- 默认不在日期面板底部放 `确定 / 清空` 按钮。清空日期范围由页面“清空条件”按钮处理；如业务需要单独清空，可在触发框右侧增加清空 icon，但不要在面板底部新增按钮区。
- 未完成区间选择时只暂存开始日期，不写入筛选值；点击面板外关闭后保持原已确认值不变。
- Axure 实现方式：用矩形/中继器生成日历日期格，用动态面板状态或全局变量保存 `range_start`、`range_end`、`range_selecting`。日期格 `OnClick` 更新变量并按条件设置选中态、区间态、hover 态。触发框显示文本读取 `range_start` 和 `range_end`。

### 4.3 下拉框

- 下拉框分为单选和多选，必须在规格说明中明确类型。二者视觉和交互不同，不要混用。
- 搜索表单中的下拉触发框默认 `200 x 32`，边框、字体、hover、focus、禁用态与输入框一致。

#### 4.3.1 单选下拉

- 单选优先使用 Axure 原生 `Droplist`。
- 默认项可以是 `请选择`、`所有状态` 或业务默认值。
- 展开后只允许选择一个选项；选择后直接在下拉框中显示选中项文本。
- 选择行为：
  - `OnSelectionChange` 后更新关联变量或中继器筛选字段。
  - 查询区单选下拉选择后不立即筛选，除非需求明确；默认点击“查询”后筛选。
  - 表单弹窗单选下拉选择后立即写入对应字段，提交时再校验。
- 清空/重置时使用 `Set selected option` 回到默认项。
- 若需要展示“请选择”但不能作为有效值，提交校验时应排除该默认项。

#### 4.3.2 多选下拉

- Axure 原生 `Droplist` 不适合作为多选控件。多选下拉使用组合控件实现：只读触发框或矩形输入框 + 下拉动态面板 + 复选框列表。
- 触发框样式：
  - 默认尺寸 `200 x 32`，白底，边框 `#D7D7D7`。
  - 右侧放下拉箭头 icon 或文本箭头。
  - hover/focus 边框高亮同输入框。
  - 触发框文本默认显示 `请选择` 或 `全部`。
- 下拉动态面板：
  - 默认隐藏，点击触发框后 `Show`，bring to front，可使用 slide down `200-500ms`。
  - 面板宽度默认与触发框一致；选项文字较长时可扩展到 `240-320px`。
  - 面板最大高度建议 `240px`，超出后开启滚动或分页。
  - 选项行高 `32px`，复选框 `13 x 13`，选项文字左对齐。
  - 第一项可为 `所有`。勾选“所有”时选中全部；取消任一子项时取消“所有”；全部子项选中时自动勾选“所有”。
  - 底部可放 `确定 / 取消 / 清空`，按钮高度 `30px`，确认在左，取消在右；搜索筛选类多选建议有“确定”，表单类多选可勾选后实时回填。
- 结果呈现：
  - 选择 0 项：显示占位 `请选择` 或业务默认 `全部`。
  - 选择 1-3 项：在触发框中用中文逗号显示选项文本，如 `中国，美国，英国`。
  - 选择超过 3 项或文本过长：显示 `已选择 N 项`，完整值保存在变量或中继器字段中。
  - 如业务要求可视化标签，标签应放在触发框下方或弹窗内容区，不撑高搜索行。
- 数据绑定：
  - 多选结果必须写入一个隐藏变量或隐藏文本元件，如 `selected_country_values`。
  - 提交或查询时读取该变量，而不是读取展示文案。
  - 中继器字段存储建议使用逗号分隔值或数组语义字符串，如 `country_codes = CN,US,GB`；展示文本可另存 `country_names`。
- 关闭行为：
  - 点击“确定”后回填展示文本并隐藏面板。
  - 点击“取消”后恢复打开前的选中值并隐藏面板。
  - 点击“清空”后取消全部选中，展示回到占位文本。
  - 点击面板外区域可隐藏面板；如已修改但未确认，按“取消”逻辑处理。

### 4.4 按钮

- 顶部/搜索按钮常高 `32px`，弹窗按钮常高 `30px`。
- 常见按钮宽度：`60px` 用于“确定/取消/新增/提交”，`90-120px` 用于“导出订单/赠送权益/同步 IAP 产品”等长文案。
- 按钮背景 `#7F7F7F`，文字白色，圆角 `5px`。主要动作可用 semibold。
- 表格/列表操作列中，除“查看 / 详情 / 预览”等纯查看类入口可使用蓝色下划线链接外，其余操作必须使用按钮样式，不使用文字按钮。常见如“编辑 / 启用 / 禁用 / 暂停 / 提交 / 状态变更”等使用灰色按钮，按钮高度建议 `26-30px`，横向间距 `8-12px`。
- 删除类操作默认不在列表操作列展示；只有需求明确提出删除能力时才增加，并必须配确认弹窗或二次确认。
- 按钮组顺序：主操作在左，取消在右。例如“提交 / 取消”“确定 / 取消”。

### 4.5 表格

- 表格使用 Axure Table 或由矩形单元格拼接。默认：
  - 表头背景 `#D7D7D7`。
  - 正文背景 `#FFFFFF`。
  - 边框 `1px #D7D7D7`。
  - 单元格文字水平居中、垂直居中。
  - 行高 `40px`。
- 常见列宽包括 `100`、`119`、`136`、`150`、`190`、`218`、`245`、`260` 等。新增列根据字段长度选择，不要让中文字段挤压换行。
- 宽表格需要区分外层列表容器宽度和表格列总宽：外层固定为页面主内容宽度，内部表格按列宽累加，超出部分由外层动态面板/滚动容器横向滚动。禁止通过放大 `body` 或画布宽度承载所有列。
- 窄表格也必须适配页面主内容宽度：当列宽总和小于列表容器宽度时，表格应拉伸到 `100%` 填满列表容器，避免右侧大面积空白；当列宽总和大于列表容器宽度时，保持列表容器固定宽度并在底部显示横向滚动条。
- 表格操作列放在右侧。查看类入口可用蓝色下划线链接；编辑、启用、禁用、暂停等操作使用灰色按钮。操作多于 2 个时优先增大操作列宽或分行按钮，避免操作区过宽。
- 分页样式可使用 `27 x 20` 小矩形，禁用文字 `#AAAAAA`，当前/可用文字 `#555555`。

### 4.6 复选框

- 复选框常用于等级、版本、包名、国家等多选场景。
- 勾选框约 `13 x 13`，边框 `#D7D7D7`。
- “所有”放在每组第一项；选择“所有”时应设计为勾选/取消同组全部项。
- 多列复选框横向间距按原型中的 57-85px 左右布置，纵向按业务分组。

## 5. 动态面板和弹窗

### 5.1 基本规则

- 所有弹窗使用 Axure `Dynamic Panel`，默认隐藏。
- 点击“查看/新增/编辑/启用/禁用/导出/赠送/补单”等操作时：
  1. `Set Panel State` 到目标状态，如 `view`、`add`、`edit`、`enable`、`disable`、`filter`、`export`。
  2. `Show` 该动态面板。
  3. 勾选 `bring to front`。
  4. 使用 `Treat as lightbox`，lightbox blur 半径使用 4。
- 纯预览弹窗可只放标题、内容和关闭 icon；确认类弹窗必须有确认和取消按钮。
- 关闭方式必须完整：关闭 icon、取消按钮、提交成功或确定后都要 `Hide` 对应动态面板。
- 弹窗白底、边框 `#D7D7D7`、圆角 `0px`。按钮圆角仍为 `5px`。

### 5.2 弹窗尺寸

- 小确认弹窗常用 `329 x 116`，居中显示，用于“确认要激活/禁用/标记过期吗？”。
- 中型新增/编辑弹窗常用 `600px` 宽，高度按内容扩展，如 `600 x 491`、`600 x 600+`。
- 列表/详情预览弹窗可用 `600 x 350`、`650 x 500` 等，保持表格完整展示。
- 弹窗内部标题使用 semibold，标题行下方放 `1px #D7D7D7` 分割线。

### 5.3 下拉式浮层

- 国家选择、多选面板等非模态浮层可用动态面板。
- 显示动效：`Show`，slide down，`500ms`，bring to front。
- 隐藏动效：`Hide`，slide up，`200ms`。
- 浮层外的确认、清空、取消按钮应同步设置文本或清空选中态。

## 6. 中继器规范

### 6.1 数据绑定

- 中继器使用 Axure `Repeater`。命名使用业务语义，如 `order_repeater`、`member_order_repeater`、`language_repeater`。
- 字段命名建议使用英文或拼音下划线，避免混乱：`order_id`、`product_id`、`status`、`created_at`。
- `OnItemLoad` 必须显式绑定字段到元件：
  - 文本元件：`Set Text on cell_order_id equal to [[Item.order_id]]`。
  - 输入框：`Set Text on input_name equal to [[Item.name]]`。
  - 下拉框：`Set Selected Option of select_status equal to [[Item.status]]`。
  - 多选下拉：按 `[[Item.field_values]]` 设置每个复选框的选中态，同时把 `[[Item.field_names]]` 或汇总文案写入触发框。
  - 复选框：按字段值设置 `selected/checked`。
- 当指定中继器字段值时，关联元件的显示值必须同步变化，不能只改静态文字。

### 6.2 增删改查

- 新增：
  1. 点击新增按钮显示 `add` 弹窗。
  2. 校验必填项。
  3. 判重或业务校验。
  4. 使用 `Add Rows` 给中继器增加一行。
  5. 隐藏弹窗并清空表单，包含输入框、单选下拉、多选下拉展示文本和隐藏变量。
- 编辑：
  1. 在行内点击编辑，记录当前行主键或直接使用当前 `Item`。
  2. 显示 `edit` 弹窗，并把 `Item` 字段写入输入框、下拉框、复选框。
  3. 提交后使用 `Update Rows` 更新对应行。
  4. 隐藏弹窗。
- 删除：
  1. 点击删除显示确认弹窗。
  2. 确定后 `Delete Rows` 或将状态字段更新为无效。
  3. 取消不变更数据，只隐藏弹窗。
- 筛选：
  1. 查询按钮读取搜索输入框和下拉框。
  2. 多选下拉读取隐藏变量或选中值集合，单选下拉读取当前选中项。
  3. 使用 `Add Filter` 或 `Remove Filter` 控制中继器数据。
  4. 清空条件按钮要清空输入、重置单选下拉、清空多选下拉、移除筛选。
- 排序/分页如有需要，使用中继器原生排序、分页和当前页设置，分页视觉遵循表格分页样式。

## 7. 常见交互模板

### 7.1 查看详情

- 行内“查看”使用蓝色下划线链接。
- OnClick：设置详情面板状态为 `view`，把当前行字段写入详情文本或表格，显示 lightbox。
- 纯预览详情只放关闭 icon；需要确认的详情保留“确定/取消”。

### 7.2 新增记录

- 顶部按钮文案：`+ 新增`。
- 弹窗标题：`新增{对象}`，字段按标签右对齐、控件右侧排列。
- 必填为空时提示 `{字段名}不能为空。`
- 添加成功后中继器新增一行，并隐藏弹窗。

### 7.3 编辑记录

- 行内按钮文案：`编辑`。
- 弹窗标题：`编辑{对象}`。
- 不可编辑字段使用禁用态输入框或纯文本。
- 提交后更新中继器当前行，并隐藏弹窗。

### 7.4 启用/禁用/删除确认

- 小弹窗文案：`确认要{操作}该项{对象}吗？`
- 按钮：`确定 / 取消`。
- 确定后更新状态或删除行；取消只隐藏弹窗。

### 7.5 查询和高级筛选

- 基础查询区直接展示常用条件。
- 条件多时使用折叠/展开动态面板，状态可命名 `default/collapse` 或 `defaullt/collapsed`，新增原型建议统一为 `default/collapsed`。
- 查询按钮通常放在当前条件组右侧或下方，和导出、新增按钮横向排列。

## 8. Axure 落地约束

- 生成 `.rp` 或 Axure 实现说明时，必须列出：
  - 页面名称和导航层级。
  - 每个动态面板的名称、状态、默认隐藏/显示。
  - 每个按钮的 OnClick 动作清单。
  - 每个中继器的数据字段、默认数据、OnItemLoad 绑定、增删改查动作。
  - 每个输入框/下拉框与中继器字段的对应关系。
  - 每个输入框的默认、hover、focus、禁用、错误状态。
  - 每个下拉框的类型：单选 `Droplist` 或多选组合控件；多选必须列出动态面板、复选框、结果展示元件和隐藏值元件。
  - 每个日期范围控件的触发框、日期动态面板、开始/结束变量、区间高亮规则、确定/清空动作。
- 不使用 Axure 不支持的 CSS Grid、React 状态、外部组件库等概念作为 `.rp` 设计依据。
- 如果只生成 HTML，则仍要模拟 Axure 的交互行为：绝对定位、动态面板显隐、lightbox、表格、中继器数据变化。

## 9. 输出验收清单

- 左侧导航存在，当前页面选中态正确。
- 面包屑、页签、标题和主内容位置与原型风格一致。
- 字体、字号、颜色、表格行高、输入框高度符合本规范。
- 输入框 hover/focus 有边框高亮，禁用和错误态清晰可见。
- 单选下拉和多选下拉样式不同，选择结果能正确展示并写入关联字段。
- 日期范围在单个触发框中展示，弹层内可选择开始/结束日期，已选区间有连续高亮，确定后能正确回填，清空后能移除值和高亮。
- 搜索标签右对齐，控件按栅格对齐。
- 表格文本默认居中，操作列清晰。
- 动态面板点击可显示，确认/取消/关闭可隐藏，lightbox 生效。
- 预览弹窗有关闭 icon；确认类弹窗有确定和取消按钮。
- 中继器新增、编辑、删除、筛选会改变数据，并同步更新关联元件。
- 所有交互都能映射到 Axure 原生动作。

## 10. HTML 预览、静态资源与缓存

- HTML 原型验收必须通过本地 HTTP 服务进行，例如从项目根目录启动服务后访问 `http://localhost:{port}/index.html`。禁止把直接双击打开的 `file://` 地址作为交付或验收入口；该方式可能导致 Chrome 未加载 CSS、JS 或加载到旧缓存。
- 每次输出 HTML 原型都要提供可直接打开的完整 HTTP 预览地址。根入口如负责跳转到子页面，也必须使用该根入口或明确提供子页面的完整 URL。
- 根入口重定向链接、HTML 中的 CSS 链接和 JS 链接必须使用同一静态资源版本号，例如 `?v=20260715-01`。修改 HTML、CSS 或 JS 后，递增该版本号并同步更新全部入口引用。
- 交付前至少检查：页面 HTML、CSS、JS 请求均成功返回；页面具有预期布局和样式；页面未依赖浏览器缓存中的旧资源。
- 如果用户反馈页面显示为无样式的原始文本，优先检查访问地址是否为 `file://`、静态资源路径是否为相对当前页面路径、CSS/JS 是否返回成功，以及页面与资源的版本号是否一致。
