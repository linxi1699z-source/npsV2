# 后台原型 Prompt 编写指南

适用于让 AI 按 `axure-pay-admin-prototype` skill 输出后台管理系统原型设计、Axure 实现说明或 Axure 风格 HTML。

## 1. 推荐 Prompt 结构

一条高质量 prompt 建议包含 8 类信息：

1. 原型目标
2. 导航位置
3. 页面模块
4. 搜索条件
5. 操作按钮
6. 表格字段
7. 表单和弹窗
8. 中继器和交互规则

不需要一次写得很长，但关键业务对象、字段、操作和数据变化要说清楚。

## 2. 通用模板

```text
请使用 /Users/xiemin/Documents/codex/原型助手/skills/axure-pay-admin-prototype 这个 skill，
按支付后台 Axure 风格，为【模块名称】设计后台原型。

页面目标：
- 这个页面用于【说明业务目标】。

导航：
- 建议放在【一级导航 / 某个一级导航下的二级页面 / 当前页面按钮跳转】。
- 如果你认为导航层级不合理，请给出更合适的层级建议。

页面需要包含：
- 搜索条件：【字段1、字段2、字段3】
- 顶部按钮：【新增、导出、批量操作等】
- 表格字段：【字段1、字段2、字段3、操作列】
- 操作列：【查看、编辑、删除、启用、禁用等】

表单 / 弹窗：
- 新增弹窗字段：【字段列表】，必填项：【字段列表】。
- 编辑弹窗字段：【字段列表】，不可编辑字段：【字段列表】。
- 查看弹窗展示：【字段或表格】。
- 确认弹窗：【删除 / 禁用 / 审核等操作的确认文案】。

下拉框和输入框：
- 单选下拉：【字段和选项】。
- 多选下拉：【字段和选项】，选择后展示规则：【直接展示 / 已选择 N 项】。
- 输入框需要包含默认、hover、focus、禁用、错误状态。

中继器：
- 表格数据使用中继器实现。
- 中继器字段：【字段名和示例值】。
- 需要支持：【新增 / 编辑 / 删除 / 筛选 / 清空条件 / 状态更新】。
- 操作后需要同步更新表格和关联输入框/下拉框。

跳转交互：
- 点击【按钮/链接】跳转到【页面】。
- 点击【查看/编辑/新增】显示对应动态面板弹窗。

请输出：
- 导航层级建议
- 页面布局说明
- 元件清单和尺寸
- 搜索区设计
- 表格设计
- 表单和动态面板状态
- 中继器字段、示例数据和交互动作
- Axure 可实现的 OnClick / OnSelectionChange / OnItemLoad 动作清单
```

## 3. 精简模板

```text
请使用项目内 axure-pay-admin-prototype skill，按支付后台 Axure 风格设计【模块名称】后台原型。

包含：
- 搜索条件：【字段列表】
- 按钮：【按钮列表】
- 表格字段：【字段列表】
- 操作：【查看、编辑、删除等】
- 弹窗：【新增、编辑、查看、确认弹窗】
- 中继器：表格数据可新增、编辑、删除、筛选，并同步输入框/下拉框值
- 跳转：【从哪些按钮跳到哪些页面】

请输出 Axure 实现规格，包含动态面板状态、表单字段、交互动作和中继器字段。
```

## 4. 示例 Prompt：退款管理

```text
请使用 /Users/xiemin/Documents/codex/原型助手/skills/axure-pay-admin-prototype 这个 skill，
按支付后台 Axure 风格设计“退款管理”后台原型。

页面目标：
- 用于查询支付订单退款记录，支持发起退款、审核退款、查看退款详情。

导航：
- 作为“订单管理”下的二级页面。

搜索条件：
- 支付订单 ID：输入框，精确查询
- 退款单 ID：输入框，精确查询
- 用户 ID：输入框，精确查询
- 退款状态：单选下拉，选项为全部、待审核、退款中、退款成功、退款失败、已拒绝
- 支付服务商：多选下拉，选项为 Google、Apple、Stripe、PayPal
- 申请时间：日期范围

按钮：
- 查询
- 清空条件
- 发起退款
- 导出退款记录

表格字段：
- 退款单 ID
- 支付订单 ID
- 用户 ID
- 产品名称
- 支付服务商
- 退款金额
- 退款状态
- 申请时间
- 更新时间
- 操作列

操作列：
- 查看：打开退款详情弹窗
- 审核：仅待审核状态显示，打开审核弹窗
- 重试：仅退款失败状态显示，打开确认弹窗

弹窗：
- 发起退款弹窗：支付订单 ID、退款金额、退款原因、备注；支付订单 ID 和退款金额必填。
- 审核弹窗：退款单 ID 不可编辑，审核结果单选下拉，审核备注输入框。
- 查看弹窗：展示退款基础信息和退款流程表格，只提供关闭 icon。
- 重试确认弹窗：文案为“确认要重新发起该笔退款吗？”，按钮为确定、取消。

中继器：
- 表格使用 refund_repeater。
- 字段包括 refund_id、payment_order_id、user_id、product_name、provider、amount、status、created_at、updated_at。
- 查询按搜索条件筛选。
- 发起退款成功后新增一行，状态为待审核。
- 审核通过后更新状态为退款中，审核拒绝后更新状态为已拒绝。
- 重试后将状态更新为退款中。

请输出 Axure 实现规格，包括页面布局、元件尺寸、动态面板状态、输入框/下拉框状态、表格和中继器绑定、所有点击交互动作。
```

## 5. 写 Prompt 时的关键点

- 明确页面是一级导航、二级页面、页签，还是从按钮跳转进入。
- 搜索条件要说明字段类型：输入框、单选下拉、多选下拉、日期范围。
- 下拉框要给选项。多选下拉要说明结果如何展示。
- 表格字段要尽量完整，尤其是状态字段和操作列。
- 弹窗要说明标题、字段、必填项、不可编辑字段、按钮和关闭方式。
- 中继器要说明字段名、示例值、增删改查和筛选后数据如何变化。
- 跳转要说明从哪个按钮或链接跳到哪个页面。
- 对状态类操作要说明状态变化，例如“禁用后状态更新为无效”。

## 6. 不推荐的 Prompt

```text
帮我做一个退款页面，要有查询、表格、弹窗。
```

问题：
- 没有导航位置。
- 没有字段。
- 没有下拉选项。
- 没有弹窗字段。
- 没有中继器数据变化。
- 没有跳转规则。

AI 只能自行猜测，输出会不稳定。

## 7. 推荐的最小信息量

如果暂时没有完整业务规则，至少给出：

```text
模块名称：
导航位置：
搜索字段：
表格字段：
操作按钮：
弹窗类型：
中继器需要支持的数据变化：
跳转页面：
```

这 8 项足够生成第一版可评审的后台原型。
